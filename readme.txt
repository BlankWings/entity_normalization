本项目的目的是实现企业，银行，监管机构等三类组织机构的全程简称归一化。
具体包含以下三个方面：
一：输入任意全称，程序输出其所有可能的简称后选。
二：输入组织机构全称和简称，检查其是否匹配。
三：输入组织机构的简称，程序在已有的全称库中进行匹配，输出匹配到的全称。

项目包含的主要文件及程序介绍：
data文件夹下包含三个文件
full_abb.txt是全称简称对文件，用于测试程序的准确率
processed_word.txt是程序对组织机构全称，通过jieba分词，自建词典和自定义规则生成的分词结果
abb_storage.txt是根据分词结果和自定义规则生成的简称侯选库

jieba_dictionary文件夹下包含两个文件，均是自建的jieba自定义词典
local.txt是常见的全国各省市名称
suffix.txt是一些机构全称中常见的词语，用于制定规则使用。

项目主要包含两个py文件
dictionary.py包含四个字典，省市简称字典，银行简称字典，监管机构简称字典和地方监管机构字典
上述四个字典用于辅助主函数制定规则。

main.py文件，是项目功能的主要实现程序。
主程序共有三个类，Process类，Ruler类和Test类
Process类用于处理全称生成对应的分词结果，输入为全称（类型为字符串），输出为分词结果（类型为字典，词性或序号为键，对应分词为值）。
Ruler类用于根据分词结果制定规则，输出最后的简称候选，输入为分词结果（类型为字典，词性或序号为键，对应分词为值），输出为简称后选（列表类型）。
Test类有四个函数分别用于测试准确率和实现上文介绍的三个项目目的。
每个类中的函数在main.py中都有详细的注释，在这里就不展开介绍了。
